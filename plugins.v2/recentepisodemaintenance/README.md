# 最近剧集维护

## 插件说明

定时维护 Jellyfin 中最近发布的剧集，用于解决追剧时 TMDb 剧集标题、简介、图片等信息更新滞后的问题。

### 1. 刷新最近剧集元数据

定时查询 Jellyfin 中最近 N 天首播的剧集，并刷新对应条目的元数据。

配置项：执行周期，最近 N 天，媒体服务器，维护媒体库，元数据刷新模式，图片刷新模式，是否覆盖已有元数据，是否覆盖已有图片。

默认不覆盖已有元数据和图片。

### 2. 重新整理最近剧集文件

定时查询 Jellyfin 中最近 N 天首播的剧集，读取当前媒体文件路径，经路径映射后交给 MoviePilot 整理链路尝试重新整理。

用于文件命名模板包含剧集标题的情况，例如新剧刚发布时文件名仍是“第 3 集”，后续 TMDb 更新正式标题后重新整理文件名。

配置项：是否启用重新整理，路径映射，试运行模式，是否跳过名称未变化的文件，单次最大处理数量。

### 3. 路径映射

媒体服务器和 MoviePilot 容器内看到的媒体路径可能不同，需要配置路径映射。

示例：

```text
/media/动画TV => /media
/media/电视剧 => /tv
```

处理规则：使用最长前缀匹配，匹配不到则跳过，映射后文件不存在则跳过并记录日志。

### 4. 试运行模式

试运行模式下只查询、识别和记录，不实际整理文件。

首次使用建议开启试运行模式，并将单次最大处理数量设置为 1。

## 安全说明

- 不直接对主视频文件执行手动重命名；
- 重新整理文件时优先调用 MoviePilot 当前整理链路；
- 目标文件已存在时默认跳过；
- 单集失败不会中断整次任务；
- 未配置路径映射或映射后文件不存在时不会处理文件。

## 推荐测试步骤

1. 只开启“刷新最近剧集元数据”，确认媒体服务器查询和刷新正常。
2. 开启“重新整理最近剧集文件”，保持试运行模式，单次最大处理数量设置为 1。
3. 检查日志中的路径映射和整理结果。
4. 确认字幕、XML 弹幕、NFO、图片等伴随文件处理符合预期后，再关闭试运行模式。
